◆ mandatory_task_period_us

const uint32* mandatory_task_period_us

mandatory_task_period_us defines the required task period for each client in microseconds. If the value is not written, the default 2000 us task period will be used. The user shouldn't change the task period after registration.

◆ mandatory_ttp_delay_us

const uint32* mandatory_ttp_delay_us

mandatory_ttp_delay_us defines the required TTP latency for a user in microseconds. Each time a new user connects the required value of the new user is checked against the already programmed ttp delay. The user shouldn't change the ttp delay while being connected. There is only one user allowed to request TTP delay

◆ mic_user_state

const mic_user_state_t* mic_user_state

mic_user_state defines if the user is interruptible or non-interruptible. The state can be changed at runtime. To inform other users about a changed state, the function MicUserUpdatedState needs to be called.

◆ num_of_mandatory_mics

uint8 num_of_mandatory_mics

Mic interface will connect to all mandatory microphones of all registered users, independent from the current use case. With that a transition between different use case can be achieved without discontinuities.

◆ use_aec_ref

const bool* use_aec_ref

use_aec_ref defines if the client requires aec_ref to be connected or not. This information can be changed by the client at runtime.
